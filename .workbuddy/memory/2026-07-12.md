- **关键环境坑**：`npx skills` CLI 不会把 skill 装进 WorkBuddy 的 `~/.workbuddy/skills`，所以在本环境（WorkBuddy）里用不了。
- 解决：把上述 9 个 skill 目录 `cp -r` 进 `~/.workbuddy/skills`。`ln -s` 在此环境报 `Operation not permitted`（无特权/未开 Developer Mode），故用复制而非软链。

## 复用要点
- 验证已装：`npx skills list -g`
- 预览仓库内含哪些 skill：`npx skills add <repo-url> -l`
- 让 skill 在 WorkBuddy 可用：复制（或有权限时软链）`~/.agents/skills/<name>` 到 `~/.workbuddy/skills/<name>`，WorkBuddy 会扫描其中的 `SKILL.md`。
- 软链特权不足时回退为 `cp -r`；缺点是后续 `npx skills update` 只更新 `~/.agents/skills` 那份，workbuddy 副本不会自动同步，需手动重拷。
